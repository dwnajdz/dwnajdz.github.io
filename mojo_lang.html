Dlaczego python jest zły?

Według firmy modular, czyli firmy odpowiedzialnej za język Mojo, Python ma znane problemy takie jak słaba wydajność niskopoziomowa i mechanizm GIL w języku CPython (wyjaśnione poniżej). Choć istnieje wiele aktywnych projektów, które mają na celu naprawę tych niedogodności, to jednak problemy te mają głębsze konsekwencje i szczególnie wpływają na dziedzine sztucznej inteligencji. A w roku 2023, czyli roku sztucznej inteligencjii te ograniczenia mają dalszy wpływ.

Global Interpreter Lock (GIL)

GIL to skrót od Global Interpreter Lock, jest to mechanizm stosowany w implementacji CPython języka Python. GIL zapewnia, że tylko jedna linia wykonywana jest w tym samym czasie w obrębie jednego procesu interpretacji Pythona, co ma na celu zapobieganie problemom z synchronizacją dostępu do zasobów współdzielonych w wielowątkowych programach. Oznacza to, że w jednym momencie tylko jeden wątek może wykonywać kod Pythona w ramach jednego procesu, co może wpłynąć negatywnie na wydajność programów wielowątkowych.

Historia Mojo 🔥

Modular zaczynał swoją działalność bez zamiaru tworzenia nowego języka programowania. Jednak podczas budowania platformy, która miała na celu zjednoczenie infrastruktury uczenia maszynowego i sztucznej inteligencji na całym świecie. Zauważono wtedy, że zaprogramowanie tego wszystkiego jest zbyt trudne. Chciano więc stworzyć innowacyjny i skalowalny model programowania, który mógłby być skierowany do akceleratorów [1] i innych systemów, powszechnych w uczeniu maszynowym. Oznaczało to potrzebę stworzenia języka programowania, który oferowałby silne metaprogramowanie [2] podczas tworzenia kodu, a także umożliwiłby zastosowanie różnych technik kompilacji, buforowanie procesu kompilacji oraz wiele innych funkcji, których brakowało w istniejących językach programowania.

akcelerator [1] - jest to specjalnie zaprojektowane urządzenie służące do przyśpieszenia obliczeń w konkretnych dziedzinach. Tutaj chodzi o akceleratory używane w uczeniu maszynowym i sztucznej inteligencji, czyli procesory graficzne (GPU) lub specjalne procesory do AI (TPU).

metaprogramowanie [2] - technika programowania, która umożliwia generowanie lub modyfikację kodu programu w czasie jego wykonania. Jest szczególnie przydatne w programach o złożonej strukturze i wykorzystywane w językach skryptowych. Metaprogramowanie jest stosowane w Pythonie i jest to jedna z cech, które czynią Python bardzo elastycznym językiem programowania.

Dlaczego Mojo?

Mojo łączy prostotę języka Python z wydajnością języków takich jak C++ czy Rust. Dodatkowo, Mojo umożliwia korzystanie z popularnych bibliotek Pythona, takich jak numpy, pandas czy matplotlib. Dzięki temu programiści mogą tworzyć czytelne i zwięzłe programy, które osiągają wysoką wydajność, a jednocześnie mają dostęp do szerokiego zakresu funkcjonalności oferowanej przez te biblioteki.

MLIR (Multi-Level Intermediate Representation), kompilator przyszłości

MLIR w prostych słowach

MLIR to narzędzie, które pomaga programistom tworzyć programy, które działają na różnych rodzajach sprzętu (np. komputery, smartfony, tablety). Działa ono na zasadzie tzw. reprezentacji pośredniej, co oznacza, że przetwarza kod programu na taki, który łatwo można dopasować do różnych rodzajów sprzętu. Dzięki temu programy działają szybciej i wydajniej na różnych urządzeniach, bez konieczności ręcznej modyfikacji kodu dla każdego z nich.

Podstawy Mojo

Są to tylko podstawy, mojo pozwala na o wiele wiele więcej (jak np. RayTracing, mnożenie macierzy, zbiór mandelbrota czy obsługe pamięci rodem z C).

Zmienne

  • let - jest zmienną stałą, co oznacza, że po przypisaniu wartości nie może być ona zmieniona.

  • var - działa podobnie jak w języku Python. Może być ona przypisana do dowolnej wartości i można ją później zmieniać. 

def your_function(a, b):
    let c = a
    var e = a
    c = b  # zwroci error: c is immutable
    e = b

    if c != b:
        let d = b
        print(d)
                            
your_function(2, 3)

Funkcje

  • def - działa podobnie jak w przypadku Pythona. Umożliwia deklarację funkcji i jej implementację. Przykład użycia: 

    def add(a, b):
    return a + b

  • fn - służy do definiowania funkcji, które wymagają określenia typu danych dla argumentów. W ten sposób możemy precyzyjnie określić, jakie typy danych powinny być przekazywane do funkcji. Na przykład: 

    fn add(a: Int, b: Int) -> Int:
    return a + b

Struktura (struct)

Typ struct w języku Mojo jest podobny do struktur znanych z innych języków programowania, takich jak C, C++ czy Go. Istotną różnicą w porównaniu do klas jest konieczność jawnego zadeklarowania wszystkich właściwości zmiennych wewnątrz struktury za pomocą deklaracji var lub let. Ta jasna deklaracja umożliwia kompilatorowi dokładne rozplanowanie i dostęp do wartości w pamięci bez konieczności korzystania z pośrednictwa. Dzięki temu struktury w Mojo mogą być efektywnie wykorzystywane do przechowywania danych i manipulacji nimi w sposób zoptymalizowany pod kątem wydajności. 

struct MyPair:
    var first: Int
    var second: Int
    
    fn __init__(inout self, first: Int, second: Int):
        self.first = first
        self.second = second   

    # funkcja ktora pozwala uzywac operatora < do porównywania zmiennych
    fn __lt__(self, rhs: MyPair) -> Bool:
        return self.first < rhs.first or
            (self.first == rhs.first and
            self.second < rhs.second)

Integracja z Pythonem

W mojo możesz importować dowolny moduł Pythona do swojego programu. 

from PythonInterface import Python

# Odpowiednik `import numpy as np` z Pythona
let np = Python.import_module("numpy")
 
# Teraz można użyc numpy tak samo jak w Pythonie
array = np.array([1, 2, 3])
print(array)

Podsumowanie

Podsumowując, Mojo może być jedną z najbardziej obiecujących nowości w świecie programowania. Dzięki prostocie i bibliotekom języka Python, programiści będą mogli tworzyć szybkie aplikacje, które będą kompatybilne z różnymi urządzeniami, dzięki kompilatorowi MLIR. To otwiera nowe możliwości dla twórców oprogramowania i może przynieść rewolucję w dziedzinie programowania.
